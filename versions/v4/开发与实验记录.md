### 阅读和审计结果

1. v1 的训练入口和测试入口：
   - 训练入口：`versions/v1/model/main.py`。
   - 训练器：`versions/v1/model/trainer.py::Trainer.train`。
   - 测试入口：同一入口加 `--only_test`，最终调用 `Trainer.evaluate`。
2. text logits 的生成函数：
   - `Trainer.predict` 中先设置 `self.model.module.set_adapter("lora_text")`。
   - 调用 `self.model(input_ids=input_ids, attention_mask=attention_mask, adapter_name="lora_text")` 得到 `scores`。
3. ID logits 的生成函数：
   - `use_gate` 分支中设置 `self.model.module.set_adapter("lora_cf")`。
   - 调用 `self.model(input_ids=item_input_ids, attention_mask=item_seq_mask, adapter_name="lora_cf", is_text=False)` 得到 `scores1`。
4. `confidence_fusion` 的文件、公式和调用位置：
   - 文件：`versions/v1/model/models/modules.py`。
   - `fusion_type="text"` 时：
     ```text
     logits_id_truth = sigmoid((logits_id - mean(logits_id)) / temperature)
     r_id = (logits_id - min(logits_id) + 1e-8) * logits_id_truth
     z_final = logits_text + alpha * r_id
     ```
   - 调用位置：`versions/v1/model/trainer.py::Trainer.predict` 和训练阶段若打开融合前损失。
5. 标签格式和训练 loss：
   - `Collator` 返回 `user_seq_data=(input_ids, attention_mask, target_ids, labels)`。
   - `labels` 是每个样本的 next item id。
   - 训练使用 `nn.CrossEntropyLoss()`，对 pooled logits 和 target ids 计算 CE。
6. validation/test 是全物品还是采样候选：
   - `Ranker` 直接在 `scores` 全维度上计算 target item rank，没有采样候选；是全物品评估。
7. checkpoint 保存与加载路径：
   - 最优模型保存到 `args.output_path/pytorch_model.bin`。
   - `evaluate` 在非 only_test 时从该路径加载。
8. 每个数据集的 `fusion_alpha` 配置来源：
   - `versions/run_common.sh` 中按数据集设置 `FUSION_ALPHA`。
   - Arts=0.7，Industrial=0.5，Musical=0.7，Prime=0.3，Video=0.5。
9. 当前结果文件和运行脚本：
   - 运行脚本：`versions/v1/run.sh`、`versions/run_common.sh`。
   - 历史结果在 `versions/v1/outputs/`。
10. 可以插入 logits 缓存和动态融合的位置：
    - 缓存位置：复用 v1 model/data/collator，在 eval forward 中分别取 text logits 和 id logits。
    - 动态融合位置：在 `confidence_fusion` 之后的残差形式上替换固定 `alpha`。
11. 数据集切分方式以及是否已有独立 calibration split：
    - 数据来自 `data/<dataset>/train.json`、`val.json`、`test.json`。
    - 当前没有独立 calibration split；v4 默认用 validation 缓存训练/校准 fuser，test 只评估。
12. Arts、Video 和小数据集的运行参数差异：

### 核心实现

代码模块：

- `versions/v4/model/utility_label.py`
  - 复现 v1 `fusion_type="text"` 下的 ID 置信残差：
    `r_id = (logits_id - min(logits_id) + 1e-8) * sigmoid((logits_id - mean(logits_id)) / temperature)`。
  - 实现 `z_final = z_text + alpha * r_id`。
  - 实现 per-sample CE、utility value 和 utility label。
- `versions/v4/model/reliability_features.py`
  - 实现 `text_entropy`、`id_entropy`、`text_margin`、`id_margin`、`branch_jsd`、`topk_overlap`、`history_length`、`history_pop_mean`、`history_pop_std`、`text_id_score_corr`。
  - 使用稳定 softmax，并对 NaN/Inf 做清理。
  - 提供基于训练集的 item popularity 统计。
- `versions/v4/model/reliability_fusion.py`
  - 实现 `FeatureStandardizer`。
  - 实现 `ContextGate`。
  - 实现残差安全参数化：
    `alpha = clamp(alpha0 + rho * tanh(gate(features)), 0, alpha_max)`。
  - 保存 fuser checkpoint 时包含标准化状态、feature schema 和 alpha 配置。
- `versions/v4/model/ranking_metrics.py`
  - 实现全物品 target rank、NDCG/Recall/MRR 和 alpha 分布统计。
- `versions/v4/scripts/cache_fusion_data.py`
  - 加载冻结 v1 checkpoint。
  - 分别前向 text/id 分支。
  - 缓存 logits、labels、user_ids、history/pop 特征、loss 和 utility label。
  - 支持 train/valid/test/calibration split。
- `versions/v4/scripts/analyze_oracle.py`
  - 实现 Oracle-Select。
  - 实现离散网格 Oracle-Alpha。
  - 输出 alpha 分布与分桶统计。
- `versions/v4/scripts/analyze_reliability.py`
  - 使用线性分类器验证 utility 可预测性。
  - 输出 AUC、Accuracy、F1、Brier、正例比例和线性系数。
- `versions/v4/scripts/train_fuser.py`
  - 只读取缓存训练 fuser，不更新主模型。
  - 损失为 `L_rank + lambda_u L_utility + lambda_r L_shrink`。
  - 根据 valid NDCG@10 保存最佳 fuser。
- `versions/v4/scripts/evaluate_v4.py`
  - 比较 text-only、id-only、v1 fixed fusion、v4 context fusion、Oracle-Select、Oracle-Alpha。
  - 输出 harm_rate、benefit_rate、average_target_rank_gain 和 alpha 统计。
- `versions/v4/tests/smoke_v4.py`
  - CPU 合成测试：alpha=0 严格退化、特征无 NaN/Inf、Oracle 可跑、fuser 保存恢复、缓存读写一致。
